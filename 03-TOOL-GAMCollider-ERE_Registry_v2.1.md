## §3 · The Two-Phase Methodology

### 3.1 Phase 1: Emergent Discovery

Phase 1 collisions ran with element beams only. The GAM Collider processed thousands of collision configurations, computing PNBA outputs from the input beams according to the empirically established fusion rules (P harmonic mean, N sum, B behavior residual, A maximum, k pair coupling). During this phase, the collision output frequently contained PNBA signatures that could not be accounted for by the input beams alone. These signatures appeared as invariants: the same PNBA four-vector emerging from independent collision configurations, without being present in any single input beam.

Each such invariant was labeled at first appearance with a symbol and provisional characterization. The labeling served two purposes. First, it permitted the corpus to reference the invariant in subsequent work without re-deriving its PNBA signature. Second, it permitted empirical observation of the invariant set's closure property.

The exploratory sweep of Phase 1 covered five to ten thousand collisions. During this sweep, new invariants appeared regularly — sometimes clustered within a specific run, sometimes distributed across runs. Toward the end of the sweep, the rate of new invariant appearance declined. After the sweep completed, new invariant appearance ceased entirely. Subsequent runs continued to produce collisions that referenced the labeled invariant set, but no new labels were required.

This closure — the empirical observation that new EREs stopped appearing after the initial sweep — is itself a structural result of the methodology. It indicates that the ERE invariant space for the PNBA collision engine, at the current substrate resolution, is finite and has been identified. The closure is not a proof of completeness in the mathematical sense; it is an empirical observation that has held across subsequent runs. Future runs at higher substrate resolutions or with novel input classes could in principle produce new invariants that would require additional entries; the corpus commits to updating the registry if this occurs, with clear timestamping of any additions to preserve the closure-history record.

### 3.2 Phase 2: Labeled Invariants as Beam Inputs

Once the registry closed, Phase 2 methodology permitted labeled EREs to be included as beam inputs in subsequent collision runs. This is the methodology used in the current OctoBeam Mode configuration of the GAM Collider v15.

The critical structural property of Phase 2 is that **the collision engine's threshold logic remains unchanged**. The engine does not treat EREs preferentially, does not weight ERE-containing beams differently, and does not apply distinct rescue logic when EREs are present. Every input beam — periodic-table element or ERE — contributes its PNBA four-vector to the fusion calculation according to the same rules that governed Phase 1. Phase classification (Noble, Locked, IVA, Shatter) is determined by the resulting τ = B/P value against the fixed threshold TL = 0.136899099984016 regardless of input class.

The consequence is that Phase 2 cannot manufacture false positives. A collision that fires as Noble in Phase 2 with an ERE input would have fired as Noble in Phase 1 with the same PNBA field configuration. The ERE input is a shortcut, not a modification. The engine will only classify a collision as coherent (Noble, Locked, or IVA) when the resulting PNBA math actually resolves within the thresholds. Combinations where the ERE input would produce PNBA field configurations outside the threshold range simply do not fire, or fire as Shatter, exactly as they would have in Phase 1.

This property matters because it establishes Phase 2 as a search acceleration technique rather than a claim generator. The ERE registry can be used as beam inputs to explore the collision space efficiently — instead of running exploratory collisions until a given ERE emerges from element combinations, the collider can be directed toward specific ERE-partner combinations of interest — while preserving the same gatekeeping that governed the exploratory phase. The output space of Phase 2 is a subset of the Phase 1 output space, not an extension of it.

The two-phase methodology has three consequences for how ERE-containing results should be interpreted in the corpus:
