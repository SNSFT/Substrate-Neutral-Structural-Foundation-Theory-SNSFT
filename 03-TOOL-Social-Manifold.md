## Technical — GitHub OAuth Flow

GitHub is the only platform with live OAuth in the current version. The flow:

1. User clicks **CONNECT GITHUB** on `/social`
2. Supabase redirects to GitHub OAuth
3. GitHub authenticates → redirects to `https://uuia.app/social` with token
4. `checkAuthCallback()` detects the session via `onAuthStateChange`
5. Engine fetches: user profile, repos, PR counts (merged/total), issue counts (open/closed)
6. `reduceGitHub()` runs the `[9,0,8,3]` formula on real numbers
7. PNBA gauges fill with live data, τ computed, phase state assigned
8. Connection saved to Supabase `social_connections` table
9. User's real IM added to the community leaderboard

**Scopes requested:** `read:user public_repo read:org` — read only, no write access.

Twitter/TikTok use demo data in the current version pending API key setup. The reduction functions are identical — real API responses are drop-in replacements for the demo objects.
